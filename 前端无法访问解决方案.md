## ⚠️ 常见问题及解决方法

### 问题1：端口被占用

**错误现象**: 启动时显示"端口已被占用"

**解决方案**:
```bash
# 方法1：终止占用端口的进程
netstat -ano | findstr :3000
taskkill /PID <进程ID> /F

netstat -ano | findstr :5000
taskkill /PID <进程ID> /F

# 方法2：修改端口（可选）
# 后端修改：编辑 backend/.env 文件，修改 PORT=5001
# 前端修改：编辑 frontend/.env 文件，添加 PORT=3001
```

### 问题2：Node命令不存在

**错误现象**: `node` 不是内部或外部命令

**解决方案**:
1. 检查Node.js安装
   ```bash
   node --version
   ```
2. 如果未安装，下载安装: https://nodejs.org/
3. 安装后重启命令行窗口

### 问题3：依赖未安装

**错误现象**: 启动时显示"模块未找到"

**解决方案**:
```bash
# 安装后端依赖
cd backend
npm install

# 安装前端依赖
cd ../frontend
npm install --legacy-peer-deps
```

### 问题4：数据库连接失败

**错误现象**: 后端启动时显示"数据库连接失败"

**解决方案**:
1. 检查MySQL服务是否运行
   ```bash
   net start mysql
   ```
2. 检查数据库配置
   ```bash
   # 测试MySQL连接
   mysql -h localhost -P 3306 -u root -p111111 -e "SHOW DATABASES;"
   ```
3. 修改后端 `.env` 文件中的数据库配置

```bash

tail -f backend/logs/combined.log

# 前端日志（在启动前端服务的命令行窗口中查看）
```

### 重置系统
```bash
# 1. 停止所有服务
taskkill /F /IM node.exe

# 2. 重启MySQL
net stop mysql
net start mysql

# 3. 清除浏览器缓存
# Chrome: Ctrl+Shift+Delete → 清除缓存数据

# 4. 重新启动服务
cd backend && node src\server.js
cd frontend && node node_modules\react-scripts\bin\react-scripts.js start
```


## 📞 技术支持

### 紧急恢复
如果以上方法都无法解决，请执行以下步骤：

1. **备份重要数据**
   ```bash
   # 备份数据库
   mysqldump -h localhost -P 3306 -u root -p111111 ninja_clan_db > backup_$(date +%Y%m%d).sql
   ```

2. **完全重置开发环境**
   ```bash
   # 停止所有服务
   taskkill /F /IM node.exe

   # 清除node_modules（谨慎）
   cd backend && rm -rf node_modules
   cd ../frontend && rm -rf node_modules

   # 重新安装依赖
   cd backend && npm install
   cd ../frontend && npm install --legacy-peer-deps

   # 重新启动
   cd backend && node src\server.js
   cd ../frontend && node node_modules\react-scripts\bin\react-scripts.js start
   ```
