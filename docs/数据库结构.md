# 《忍者必须死3》家族网站数据库结构

## 📊 数据库概览

### 数据库信息
- **名称**: `ninja_clan_db`
- **字符集**: `utf8mb4`
- **排序规则**: `utf8mb4_unicode_ci`
- **时区**: `+08:00` (中国时区)

### 表数量
共 **10个** 核心数据表，按功能模块组织。

## 👥 用户管理模块

### 表: `users` (用户表)
**描述**: 存储所有家族成员信息

| 字段名 | 类型 | 约束 | 描述 |
|--------|------|------|------|
| `id` | INT | PK, AI | 用户ID |
| `game_id` | VARCHAR(50) | UNIQUE, NOT NULL | 游戏内ID |
| `username` | VARCHAR(50) | UNIQUE, NOT NULL | 登录用户名 |
| `password_hash` | VARCHAR(255) | NOT NULL | 密码哈希 |
| `display_name` | VARCHAR(50) | NOT NULL | 显示名称 |
| `email` | VARCHAR(100) | NULL | 邮箱地址 |
| `avatar_url` | VARCHAR(255) | NULL | 头像URL |
| `role` | ENUM('member','captain','admin') | DEFAULT 'member' | 用户角色 |
| `clan_rank` | INT | DEFAULT 0 | 家族排名 |
| `status` | ENUM('active','suspended','inactive') | DEFAULT 'active' | 用户状态 |
| `last_login` | TIMESTAMP | NULL | 最后登录时间 |
| `created_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引**:
- `PRIMARY KEY` (`id`)
- `UNIQUE KEY` (`game_id`)
- `UNIQUE KEY` (`username`)
- `INDEX` (`role`)
- `INDEX` (`status`)

**数据示例**:
```sql
INSERT INTO users VALUES (
  1, '200801034671', '夏投岚', '$2a$10$...', '夏投岚',
  NULL, NULL, 'member', 0, 'active',
  '2026-03-28 16:20:18', '2026-03-28 14:02:34', '2026-03-28 16:20:18'
);
```

## 🗃️ 数据库初始化脚本

### 完整创建脚本
```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS ninja_clan_db
CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE ninja_clan_db;

-- 设置时区
SET time_zone = '+08:00';

-- 创建 users 表
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    game_id VARCHAR(50) UNIQUE NOT NULL,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    display_name VARCHAR(50) NOT NULL,
    email VARCHAR(100),
    avatar_url VARCHAR(255),
    role ENUM('member', 'captain', 'admin') DEFAULT 'member',
    clan_rank INT DEFAULT 0,
    status ENUM('active', 'suspended', 'inactive') DEFAULT 'active',
    last_login TIMESTAMP NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_role (role),
    INDEX idx_status (status)
);

-- 创建 abyss_teams 表
CREATE TABLE abyss_teams (
    id INT AUTO_INCREMENT PRIMARY KEY,
    team_number TINYINT UNIQUE NOT NULL,
    team_name VARCHAR(50) NOT NULL,
    captain_id INT,
    status ENUM('active', 'inactive') DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (captain_id) REFERENCES users(id) ON DELETE SET NULL
);
```

## 🔍 数据查询示例

### 查询用户及其深渊队伍
```sql
SELECT
    u.username,
    u.display_name,
    u.role,
    at.team_name,
    at.team_number
FROM users u
LEFT JOIN abyss_teams at ON u.id = at.captain_id
WHERE u.status = 'active'
ORDER BY u.clan_rank;
```

### 查询深渊队伍成员
```sql
SELECT
    at.team_number,
    at.team_name,
    u.username,
    u.display_name,
    u.clan_rank
FROM abyss_teams at
LEFT JOIN users u ON at.captain_id = u.id
ORDER BY at.team_number;
```

### 统计用户活跃度
```sql
SELECT
    DATE(last_login) as login_date,
    COUNT(*) as active_users
FROM users
WHERE last_login >= DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY DATE(last_login)
ORDER BY login_date DESC;
```
